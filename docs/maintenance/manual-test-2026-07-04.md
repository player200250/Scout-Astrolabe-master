## 發現的問題（待處理）

### ❓ D1 — 主頁儀表板/白板雙模式不直覺
- **現象**：主頁同時有「儀表板模式」與「白板模式」，使用者覺得兩者的切換與用途不直覺。
- **待討論**：兩模式的定位與切換方式（是否預設、如何切換、是否合併或明確分工）。
- **狀態**：需討論

### 💬 D2 — 右鍵選單無規範文件（功能齊全但不可見）
- **現象**：右鍵選單功能其實完整，但散在 `src/utils/contextMenuUtils.tsx`，無使用者可見的功能說明，使用者不知道有哪些。
- **已整理出的完整規範**：
  - **A. 空白處右鍵（建立卡片）**：📝文字／✅待辦／🔗連結／🖼️圖片／A標題／📌便利貼(子選單:顏色)／▦表格(子選單:2/3/4欄)／🎨顏色樣本／📎上傳檔案(僅 Electron)／📋從模板新增(空白+5內建模板:會議記錄/讀書筆記/問題拆解/目標設定/想法捕捉+自訂模板+刪除自訂模板)
  - **B. 單張卡片右鍵（通用）**：🔍縮放至此卡片／📋複製卡片／📦移到白板…／🗑️刪除卡片／選單底部顏色選擇器(便利貼專屬色盤)
  - **B-類型限定**：連結卡→✏️編輯連結；表格卡→☑/☐開關標題列、▦欄數(2/3/4,減欄有資料先確認)；文字卡→⭐存為模板
  - **C. 多選(≥2)批次**：📋複製N張／⬛對齊(靠左/置中/靠右/靠上/靠下,≥3張加水平/垂直均分)／📊批次狀態／⚑批次優先級／🏷批次標籤／📦移動N張／🗑️刪除N張
  - 側邊欄(`[data-sidebar]`)上右鍵會被忽略。
- **建議**：把此規範寫成正式文件（如 `docs/context-menu-spec.md`），並考慮 App 內加一處「右鍵功能」提示。
- **狀態**：需討論（要不要建文件 / 加 App 內提示）

### ⚠️ D3 — 白板預覽卡片無法區分主板/子板
- **現象**：所有白板的預覽卡片（縮圖）長得一樣，主板與子板視覺上無區別，看不出層級關係。
- **技術可行性**：`BoardRecord` 已有 `parentId` 欄位（子板才有值），可據此加辨識。
- **可能改法（待討論）**：子板加「↳ 父板名稱」標籤 / 主板加層級 badge / 縮排或分組顯示 / 子板角標圖示。
- **狀態**：待處理（UX 改善，低風險）

### 💬 D4 — 側邊欄拖拽功能（用語校正 + 需求釐清）
- **使用者原述**：希望「最近使用」的部分能拖拽。
- **現況校正**（`BoardTabBar.tsx`）：
  - **最近使用**：依 `lastVisitedAt` 自動排序、取最近 5 筆的**衍生清單**，每次造訪重排 → **拖排序語意不成立**（會被下次造訪覆蓋）。
  - **未分類**：**已支援拖曳排序**（`handleDragEnd` → `onReorderBoards`，`@dnd-kit`）。
  - **資料夾歸類**：目前靠**右鍵選單「📁 移入資料夾…」**，**不能用拖的**。
- **可討論方案**：
  - **A（推測為真實需求，🔴 高）**：拖白板→資料夾放開即歸類（把右鍵歸類升級為拖放）。
  - **B（🟡 中）**：擴大拖排序範圍至釘選/資料夾內（目前僅未分類可）。
  - **C（🟡 中）**：允許從「最近使用」拖出去歸類/釘選，當快速入口。
- **待定細節**：拖放高亮回饋、是否支援多選拖曳、拖到資料夾標題 or 展開區。
- **狀態**：需討論（先確認要 A/B/C 哪個方向）

### ⚠️ D5 — 卡片庫辨識度不足（列表檢視 UI 需調整）
- **現象**：卡片庫難以辨識各卡片類型，文字卡「一整片都是字」看不清楚。
- **根因**（`CardLibrary.tsx`）：
  - **列表檢視 `renderListCard`**：只有 36px 表情圖示（📝）當類型標記，**無文字標籤**，右側全是內容預覽。
  - **格狀檢視 `renderGridCard`**：其實已有「圖示 + 文字標籤」，辨識度較佳（`TYPE_ICON` + `TYPE_LABEL` 已存在，line 37-38）。
  - 類型圖示背景所有類型**同一中性灰**，無顏色區分。
- **具體可調整項**：
  1. 列表檢視圖示旁補上類型文字標籤（格狀已有，補齊）。
  2. 類型圖示依類型上色（📝藍/✅綠/🔗紫…），一眼分類。
  3. 文字卡標題（H1/首行）用粗體當標頭、內文另起一行，打破整片文字感。
- **備註**：使用者另提「有些 UI 介面需要調整」較概括，待補具體項目。
- **狀態**：待處理（UX 改善，低風險）

### ⚠️ D6 — 大量物件一律橫向排列（應改網格換行）
- **現象**：批次搬卡到白板、或建立多張卡片時，卡片一律橫向無限往右排，數量多時拉成一條長龍。
- **根因**（`src/hooks/useInboxCards.ts`）：
  - `handleMoveCardsToBoard`（line 52-64）：`y` 固定 100，`x += w + 40` 逐張右移，無換行。
  - `handleAddCardToInbox`（快速捕捉）同樣「append 在最右、`y:100`」→ 收件匣用久變超長橫列。
- **建議改法**：改網格（grid）換行——排到一定寬度/欄數就 `y += rowHeight`、`x` 歸位。欄數可固定（如 5 欄）或 `Math.ceil(Math.sqrt(n))` 讓大量卡片接近方形。
- **影響範圍**：`handleMoveCardsToBoard`（必要）、`handleAddCardToInbox`（可選）；未來「圖片集體上傳」也應套用。
- **狀態**：待處理（UX 改善，低風險）

### 💬 D7 — 任務中心 / 復盤中心使用率偏低
- **現象**：兩個面板很少被打開使用，價值未被感受到。
- **現況定位**：任務中心＝待辦的「死線視角」（跨白板彙整 todo + 逾期）；復盤中心＝週回顧 + 日曆/日記檢視（AI 草稿生成為 roadmap AI-4 規劃中）。
- **待討論方向（提升 engagement 的候選）**：
  1. **主動浮現，而非被動查看**：側邊欄「任務中心」入口顯示逾期/今日到期數量 badge；有逾期時主頁頂部提示（已有 `overdueBanner` 面板可強化）。
  2. **降低進入門檻**：主頁儀表板直接嵌入「今日任務 / 本週待復盤」小工具，不用特地開面板（呼應 D1 主頁定位討論）。
  3. **養成習慣的觸發點**：每日/每週固定提醒（週日晚提示復盤）；完成待辦時給即時回饋。
  4. **讓復盤有料**：優先推進 AI-4（週回顧 AI 草稿），把「從零寫」變成「改 AI 草稿」，降低復盤惰性。
  5. **反思是否真需要**：若自用情境確實用不到，考慮簡化或合併，而非硬提升（呼應 avoid-overoptimization）。
- **狀態**：需討論（先確認要提升，還是簡化/合併）

### ⚠️ D8 — 筆刷工具卡頓（需實測確認根因）
- **現象**：使用筆刷繪製時明顯卡頓/慢。
- **已排除（非 in-stroke 卡頓來源）**：
  - 存檔：`WhiteboardTools.tsx:653` `saveDebounce` 每次變更僅 `clearTimeout`+`setTimeout(500ms)`，畫圖時持續重設，不會在筆畫中觸發。
  - 縮圖：`exportToBlob` 整板 PNG 昂貴，但已「大板(>150 shape)跳過 + 小板最多 15 秒一次」（`constants.ts:15-17`）。
  - 回收站 undo 監聽（`line 300`）：只讀 `change.changes.added`，畫圖多為 update，幾乎空跑。
- **可疑點（未實測）**：筆刷為 tldraw 原生工具本應夠快；最可能是白板上的**重型卡片元件**（Tiptap 富文字 / base64 圖片 / 連結預覽）在 draw shape ~60fps 更新時持續重繪，畫布上卡片越多/越重每幀成本越高（與 TD-IMG／OOM 同族病根）。
- **下一步**：DevTools Performance 錄一段筆刷，確認卡在 tldraw render 或某卡片元件重繪。
- **可能治法**：畫圖時凍結/降級卡片內容渲染；或減輕卡片成本（TD-IMG 圖片移出 base64）。
- **狀態**：待實測（不亂猜 fix，先量測）
