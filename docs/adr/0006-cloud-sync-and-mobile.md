# ADR 0006：選擇性雲端同步與行動端（PWA）

## 狀態

已採用（2026-06-21）—— **修正（amends）** [0005 本機優先產品方向](0005-local-first-product-direction.md)，但不推翻其本機核心原則。

---

## 背景

ADR 0005 確立「本機優先、不設後端、不做雲端同步」。但實際使用揭露一個 0005 未解決的需求：

> 使用者人在外面、只帶手機，而軟體與資料都在桌機，**完全無法存取**。

關鍵限定是「**在外面**」。0005 的「未來演進方向」曾列出 LAN/P2P 同步，但 LAN Sync 只在同一區域網路有效，無法滿足外網存取。要在外網看到桌機資料，資料**必須**經過網際網路——這與 0005「不上雲」的字面承諾衝突。

同時，本專案的定位已釐清為 **自用工具**（非商業產品）。這使 0005「隱私 = 不上雲」的論證前提弱化：自用情境下，資料上雲的顧慮（第三方存取、商業資料外洩）大幅降低，而「桌機需常開」反而是更實際的痛點。

---

## 決策

採用**選擇性雲端同步 + 行動端 PWA**，但保留本機優先的核心：

1. **本機 IndexedDB 仍是 ground truth**；雲端是**同步層**，非主要儲存。
2. **雲端同步預設可選**（非強制）：不啟用時，App 行為與 0005 完全相同（純本機、離線、免費）。
3. **行動端為 PWA**，能力限定「看全部 + 簡單編輯（文字/待辦/標籤/狀態/優先級）」，**不含畫布**。不做 React Native 原生客戶端。
4. **後端採 Supabase**；衝突策略初版為整板 last-write-wins（比 `updatedAt`），同步前自動備份兜底。

詳細階段與技術方案見 [roadmap-mobile.md](../roadmap-mobile.md)。

---

## 後果

### 正面

- 解決核心痛點：在外網用手機即可檢視/速記/簡單編輯
- 桌機無需常開（相對於 Tailscale 方案）
- 不啟用同步時，0005 的所有正面性質（隱私/離線/免費/速度）完整保留
- PWA 與桌機共用同一 repo 與 snapshot 工具，順帶完成 P1 平台抽象與 X2 Web 載體

### 負面 / 取捨

- **隱私讓步**：啟用同步時資料離開本機。緩解：自用情境已接受；同步為可選；本機仍為 ground truth
- **複雜度上升**：引入認證、同步佇列、衝突解決——正是 0005 當初為求簡單而避開的
- **整板 LWW 風險**：兩端同時改同板會丟資料。緩解：同步前自動備份
- **行動端能力受限**：`file` 卡無法跨裝置開啟、`image` 卡 base64 體積問題（初版限制）

---

## 與 0005 的關係

- 0005 的**本機核心原則不變**：IndexedDB 為 ground truth、離線完整可用、不強制帳號。
- 修正之處：0005「不設計任何後端或雲端同步」一條，改為「**可選的雲端同步層**」，對應 0005 自身「未來演進方向」的第 2、3 點（選擇性雲端備份、保持本地核心）。
- 0005「不建議方向」中的「強制雲端帳號」「中央化資料儲存」**仍不採用**——同步是可選的附加層。
